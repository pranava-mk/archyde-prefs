# apps workspace

## What this is for
Tracks per-application user configuration files that live under `~/.config/`. These are hand-tuned user preferences that are not regenerated by HyDE or wallbash and should survive theme changes.

## Routing table

| Task | Read | Skip |
|---|---|---|
| Tune a tracked app's settings | matching file above | excluded files |
| Add a new app config | check exclusion table first | wallbash/HyDE-managed paths |
| Investigate why a config keeps resetting | exclusion table (likely HyDE/wallbash-managed) | — |

## Conventions

- Only track files that are entirely user-authored and not overwritten by HyDE or wallbash on theme change.
- Downloaded assets (themes, plugins with embedded API keys) belong on the new machine's download step, not in this repo.
- Runtime state (clipboard history, logs) is never tracked.
